## AI 부동산 스캐너: MVP 제품 요구사항 정의서 (PRD)

---

### 1. 제품 개요

#### 1.1. 제품명
AI 부동산 스캐너

#### 1.2. 핵심 가치
AI의 신속한 법률 답변과 현직 변호사의 전문적인 검토를 결합하여, 복잡하고 어려운 부동산 법률 문제에 대한 가장 빠르고 신뢰도 높은 해결책을 제공합니다.

#### 1.3. MVP 목표
신속한 개발 이터레이션(iteration)을 통해 MVP(최소 기능 제품)를 완성하고, 내부 베타 테스트를 성공적으로 수행하여 핵심 기능의 안정성과 사용성을 검증합니다. 이를 통해 정식 출시 전 사용자의 핵심 피드백을 확보하고 제품을 개선합니다.

#### 1.4. 성공 지표
*   핵심 기능 완료율: 내부 베타 테스트 참여자들이 핵심 기능(AI 질문, 변호사 답변 작성, 포인트 충전)을 95% 이상 성공적으로 완료합니다.
*   서비스 안정성: 테스트 기간(2주) 내 서비스 중단을 유발하는 치명적인 버그(Critical Bug) 0건을 목표로 합니다.
*   사용자 만족도: 테스트 참여자 대상 설문조사에서 서비스의 핵심 가치 및 사용 편의성에 대한 만족도 5점 만점에 4점 이상을 달성합니다.

### 3. 포함 페이지 (Scope)

*   공용 레이아웃: Header (서비스 명, AI 빠른상담, 변호사 전문가 답변, 마이페이지)
*   메인 홈 페이지 (`/`): 서비스 소개 및 핵심 기능으로 안내하는 랜딩 페이지
*   회원가입/로그인 페이지 (`/sign-up`, `/log-in`): 일반/변호사 회원 유형 분리
*   AI 빠른상담 페이지 (`/ai-qna`): Gemini API 연동 채팅 UI
*   변호사 전문가 답변 페이지 (`/lawyer-board`): 공개된 질문 목록 게시판
*   질문 상세 보기 페이지 (`/qna/{questionId}`): 질문, AI 답변, 변호사 답변 통합 뷰
*   마이페이지 (`/my-page`): 사용자 유형별 기능 제공 (질문/답변 내역, 프로필, 포인트 관리 등)
*   변호사 공개 프로필 페이지 (`/lawyer/{lawyerId}`): 변호사 상세 정보 및 전문성 어필
*   관리자 페이지 (`/admin`): 변호사 승인 및 신고 관리를 위한 내부 페이지

### 4. 사용자 여정 (User Journey)

#### 4.1. 타겟 유저: 일반 실거주자 (30-50대)
*   목표: 전세 계약 연장 시, 임대인이 제시한 새로운 특약사항이 법적으로 문제없는지 빠르게 확인하고 싶다.
*   여정:
    1.  방문 (`/`): 검색을 통해 'AI 부동산 스캐너' 랜딩 페이지에 접속하여 서비스 소개를 확인한다.
    2.  회원가입 (`/sign-up`): 'AI에게 바로 질문하기' 버튼을 클릭, 카카오 소셜 로그인을 통해 3초 만에 간편하게 가입한다.
    3.  AI 상담 (`/ai-qna`): 로그인 후 이동된 AI 빠른상담 페이지에서 특약사항 내용을 입력하고 AI의 1차 답변을 즉시 확인한다.
    4.  질문 공개: 더 전문적인 검토를 위해 '이 질문을 변호사에게 공개' 체크박스를 선택한 채 대화를 종료한다.
    5.  변호사 답변 확인 (`/qna/{questionId}`): 다음 날, 변호사 답변 등록 알림을 받고 질문 상세 페이지로 이동하여 답변 내용을 확인한다.
    6.  답변 채택: 가장 명쾌한 답변을 제공한 변호사의 답변을 '채택'하여 감사를 표한다.
    7.  내역 관리 (`/my-page`): 마이페이지의 '나의 질문 내역'에서 과거 질문과 답변을 다시 확인한다.

#### 4.2. 타겟 유저: 변호사
*   목표: 합리적인 비용으로 부동산 법률 분야의 잠재 고객을 확보하고 전문성을 홍보하고 싶다.
*   여정:
    1.  방문 및 가입 (`/`, `/sign-up?type=lawyer`): 법률 커뮤니티를 통해 서비스를 인지하고, 랜딩 페이지의 '파트너 변호사로 활동하기' 버튼을 통해 변호사 회원으로 가입한다.
    2.  자격 인증 (`/my-page`): 가입 완료 후 마이페이지의 '변호사 인증 관리'에서 변호사 자격 증명 서류(.pdf)를 업로드하고 승인을 기다린다.
    3.  포인트 충전 (`/my-page`): 관리자 승인 완료 후, '포인트 관리' 메뉴에서 Toss Payments를 이용해 답변 활동에 필요한 50,000 포인트를 충전한다.
    4.  질문 탐색 (`/lawyer-board`): 변호사 전문가 답변 페이지에서 자신의 전문 분야와 관련된 최신 질문들을 탐색한다.
    5.  답변 작성 (`/qna/{questionId}`): 답변할 질문을 클릭하여 상세 내용을 확인한 후, 1,000 포인트를 지불하고 의뢰인에게 신뢰를 줄 수 있는 전문적인 답변을 작성한다.
    6.  프로필 노출 (`/lawyer/{lawyerId}`): 질문자가 자신의 프로필을 클릭하여 상세 정보를 확인할 것을 기대하며, 잠재적인 유료 자문으로 이어지기를 희망한다.

#### 4.3. 타겟 유저: 서비스 관리자
*   목표: 변호사 회원의 자격을 신속하게 검증하고, 서비스 내 콘텐츠의 건전성을 유지한다.
*   여정:
    1.  로그인 (`/admin`): 관리자 계정으로 관리자 페이지에 로그인한다.
    2.  변호사 승인 관리: '변호사 승인 관리' 탭에서 새로 가입을 신청한 변호사 목록과 제출 서류를 확인하고, 검증 절차를 거쳐 '승인' 처리한다.
    3.  신고 관리: '신고 관리' 탭에서 사용자들이 신고한 부적절한 답변 내용을 검토하고, 정책에 따라 해당 답변을 블라인드 처리한다.
